## **Learning Objectives**
By the end of this week, students will be able to:
1.  Identify where variables live in memory (Data Section vs. Stack).
2.  Use `sizeof()` to verify variable footprints on a 16-bit architecture.
3.  Implement robust code that handles variable overflow and underflow.
4.  Apply fixed-width data types (`uint8_t`, `int16_t`, etc.) for portable firmware development.
5.  Use `typedef` to create descriptive and maintainable custom types.
